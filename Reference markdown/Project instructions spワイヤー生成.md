## STEP 3 — ユーザーに確認すべき判断分岐の例

以下のような場合は、HTMLを生成する前にユーザーに選択肢を提示します。
選択肢は1〜2問に絞り、箇条書きで短く提示します。

**確認が必要な典型パターン：**

| 状況 | 提示する選択肢の例 |
|---|---|
| 比較表のSP変換 | A) カード型 / B) 横スクロールテーブル |
| フィルターボタン5択以上 | A) 横スクロール（`overflow-x: auto`） / B) 2行折り返し |
| サイドバーのSP配置 | A) 本文直後に移動 / B) 関連記事の前に挿入 |
| 長いリストの表示 | A) 全表示 / B) 折りたたみ（`<details>`） |
| ページヘッダーのCTA数 | A) 主CTAのみ / B) 2択維持 |

確認不要な場合（ルールが明確な場合）は即座に生成に進みます。

## 変換ルール（絶対遵守）

`03_SPワイヤー専用ルール.md` が正本です。以下はその主要点の抜粋。

### レイアウト
- PCの横並びは原則1カラム縦積みに変換する（§3-2）
- ファーストビューの順序: ラベル → h1 → 説明文 → 主CTA → 副CTA → ビジュアル（§3-3）
- 2カラム維持が許容されるのは「数字が短い実績カード」「小アイコン付き要約カード」「画像サムネイル一覧」のみ（§3-2）

### タップターゲット
- 主要ボタン・メニュー・アイコンボタンは **44×44px以上**（§2-1）
- 隣接ターゲット同士は **8px以上** 離す（§2-1）

### ハンバーガーメニュー
- トリガーは必ず `<button>`（`div` や `a` は禁止）（§2-2, §7-5）
- `aria-expanded` / `aria-controls` を必ず付ける
- Escape で閉じる前提で設計する
- PC/SP で DOM を二重化しない

### フォーム（§3-6）
- 1カラム必須
- ラベルは `<label>` で明示（プレースホルダーのみで代替禁止）
- 送信ボタンは 44px 以上・幅100%

### CSSルール
- テキスト領域に固定 `height` を使わない（§7-2）
- 画像ラッパーは `aspect-ratio` + `object-fit: cover`（§2-3）
- `100vh` を本文レイアウトに使わない。全画面演出のみ `svh`/`dvh`（§2-4）
- インライン `style` 属性を新規で使わない（§7-4）

### WordPress実装を見越した構造
- `<main>` 直下のセクションはフラットに並べる（§6-1）
- セクション順序・削除要素・ループ対象をコメントで明記する
- ワイヤー専用クラス（`.ph-*`）は使用可だが削除対象として注記する（§5-5）

### 命名規則（§5）
- プロジェクトプレフィックス: `mf-`
- 構造: BEM（`mf-block__element--modifier`）
- 状態: `is-open` / `is-active`
- JSフック: `js-*`（CSSを当てない）
- 禁止クラス: `.title` `.text` `.inner` `.wrap` `.item` `.btn` `.active` `.open`

### SP専用デザイントークン（§4）
- 左右余白: `16px`（最大 `20px`）
- セクション上下余白: `48px` / `56px` / `64px` の3段階
- 本文フォント: `16px / line-height 1.7`
- 補足文: `14px / line-height 1.6`
- セクション見出し: `24〜28px`（`clamp()` 推奨）
- ヒーロー h1: `28〜32px`
- ボタン文字: `16px以上`
- ブレークポイント: `@media (max-width: 768px)`（`--bp-sp: 768px`）

## 出力フォーマット

### ① SPワイヤーHTML（メイン成果物）

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ページタイトル — More Friends</title>
  <link rel="stylesheet" href="wireframe-base.css">
  <!-- ページ固有CSS: sp-[pagename].css -->
  <style>
    /* SP専用スタイル（このファイルに仮置き） */
    /* 本番では sp-[pagename].css として分離する */
  </style>
</head>
<body>
  <!-- ========================================
    header
    parts/header.php
    編集区分: 固定
  ========================================= -->
  <header class="mf-header">
    ...
  </header>

  <main>
    <!-- ========================================
      [セクション名]
      parts/section/[partname].php
      編集区分: [固定 / 編集可 / CMSループ]
      SP変換: [変換内容の1行メモ]
    ========================================= -->
    <section class="mf-[section-name]">
      ...
    </section>
  </main>

  <footer class="mf-footer">
    ...
  </footer>
</body>
</html>
```

コメントのルール:
- 各セクションの開始コメントに `parts/` パスと編集区分を必ず記載する
- ⚠️注意点があった箇所は `<!-- ⚠️ TODO: ... -->` コメントを残す
- ワイヤー専用クラスは `<!-- ワイヤー専用: 本番前に削除 -->` を付ける

## 禁止事項

- PCワイヤーをそのまま縮小して出力すること
- 1回の作業で複数ページを同時生成すること
- インライン `style` 属性を新規追加すること
- `div` や `a` でハンバーガーボタンを実装すること
- `.title` `.inner` `.wrap` などの汎用クラスを新規使用すること
- ルール根拠なしに PC レイアウトの構造（横並び等）を SP でも維持すること
- 引き継ぎメモを省略すること
